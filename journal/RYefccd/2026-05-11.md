==[HAProxy](https://www.haproxy.com/glossary/what-is-the-model-context-protocol-mcp)可以作为模型上下文协议 (MCP)服务器的高性能负载均衡器和反向代理。==这些技术正日益广泛地用于将人工智能模型与工具、数据库和其他代理连接起来。截至 2025-2026 年，HAProxy 将用于扩展 MCP 服务，其方法是将流量分配到多个后端服务器，利用`mcp-session-id`标头确保会话持久性，并在边缘验证协议特定的要求。 [[1](https://www.haproxy.com/glossary/what-is-the-model-context-protocol-mcp), [2](https://thenewstack.io/scaling-ai-interactions-how-to-load-balance-streamable-mcp/)]

MCP/A2A 生态系统中的 HAProxy [[1](https://agentgateway.dev/docs/standalone/main/tutorials/a2a/)]

- **负载均衡 MCP/A2A：** HAProxy 管理 MCP 和 A2A 使用的长期双向通信会话（Streamable HTTP）的流量，这对智能 AI 至关重要。
- **会话持久性：** HAProxy 会话保持表确保来自同一会话上下文的客户端请求被路由到同一后端 MCP 服务器。
- **安全性和验证：** HAProxy 可以检查`Accept`标头并在边缘验证 MCP 特定规则，以保护后端代理基础架构。
- **A2A（代理到代理）：** MCP 将 AI 与工具连接起来，而 A2A 则处理代理之间的通信。与 MCP 类似，A2A 使用基于 HTTP 的 JSON-RPC，HAProxy 可以代理该请求。 [[1](https://konghq.com/blog/product-releases/securing-observing-governing-mcp-servers-with-ai-gateway), [2](https://blog.mygraphql.com/en/posts/ai/ai-devops/agent-gateway/agentgateway-impl/), [3](https://www.gravitee.io/blog/googles-agent-to-agent-a2a-and-anthropics-model-context-protocol-mcp), [4](https://github.com/envoyproxy/envoy/issues/39174), [5](https://thenewstack.io/scaling-ai-interactions-how-to-load-balance-streamable-mcp/)]

关键解决方案和工具

- **[Agentgateway（开源）](https://github.com/agentgateway/agentgateway)：**一个专门的代理，为代理到 LLM/A2A 场景提供安全性、可观察性和治理。
- **[Gravitee](https://documentation.gravitee.io/apim/ai-agent-management/create-an-a2a-proxy)：**提供企业级 A2A 代理支持。
- **[Kong Gateway](https://konghq.com/blog/engineering/ai-gateway-mcp-gateway-mcp-server-breakdown)：**提供对 AI Gateway 和 MCP/A2A 流量的原生支持，支持用于安全和速率限制的插件。
- **[Context Forge（IBM）](https://github.com/IBM/mcp-context-forge)：**充当 MCP/A2A 的网关、注册表和代理，促进工具联合。 [[1](https://github.com/IBM/mcp-context-forge), [2](https://github.com/agentgateway/agentgateway), [3](https://agentgateway.dev/docs/standalone/latest/faqs/), [4](https://documentation.gravitee.io/apim/ai-agent-management/create-an-a2a-proxy), [5](https://github.com/Kong/guidance-for-kong-genai-mcp-and-a2a-gateways-on-aws), [6](https://developer.konghq.com/ai-gateway/), [7](https://heidloff.net/article/context-forge-client/)]

实施细节

- **配置：** MCP 的 HAProxy 配置涉及使用 HTTP/2 或 HTTP/3 来处理每个连接中的多个流，因为 MCP 需要双向数据流。
- **路由：**可以使用 HAProxy 中的 ACL，根据请求的工具或服务将流量路由到不同的 MCP 服务器。
- **示例设置：**设置通常包括`mcp-gateway-registry`注册代理和服务器，使用代理（如 HAProxy 或专用代理网关）根据这些注册信息路由流量。 [[1](https://github.com/agentic-community/mcp-gateway-registry), [2](https://generativeai.pub/mcp-servers-explained-python-and-agentic-ai-tool-integration-aa2ddca6cbe5), [3](https://www.haproxy.com/blog/http-keep-alive-pipelining-multiplexing-and-connection-pooling), [4](https://www.haproxy.com/glossary/what-is-the-model-context-protocol-mcp), [5](https://thenewstack.io/scaling-ai-interactions-how-to-load-balance-streamable-mcp/)]
