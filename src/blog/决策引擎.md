
# 决策引擎

> 决策引擎: 将计算, 决策(判断), 配置分离, 可视化编排决策流程, 对业务场景在一段时空上进行访问控制.

> 随着AI从“工具”走向“决策中枢”, 决策系统需要把数据、算法与业务动作串联成闭环的运营系统
## 背景

在多年的安全研发和业务运营中，为了快速去拥抱变化，在业务发展初期，基本都是通过硬编码的方式将风控逻辑部署在业务逻辑中实现，但随着业务领域扩大，运营复杂度的提升，各种合规需求的适配，导致硬编码方式出现了策略分散无法管理、逻辑同业务强耦合、策略更新迭代率受限于开发、对接成本高等多种问题。

想象一下下列场景:
1. 在营销场景中, 业务改一条风控阈值、调整优惠券发放条件，需求设计, 开发编码, 测试验收, 发版上线，周期数天甚至几周。
2. 为了应对紧急的业务安全，技术需要快速上线防御垫片逻辑，长此以往相似判断逻辑散落在前端、网关、微服务、定时任务，多份代码副本，修改极易漏改，出现判定不一致、线上漏洞。
3. 

随着业务的发展，技术的迭代，运营规则和业务安全之间的界限变得越来越模糊。

此时，我们需要有一套可配置化的规则管理平台，进而实现风控策略与业务解耦、快速部署、验证。所以，极验公司将验证码中的规则引擎部分抽取出来，结合高性能的实时窗口计算相关访问资源主体的频率，聚集程度和关联关系，集合动态配置功能，**将计算，决策(判断)，配置分离，可视化编排决策流程，对业务场景进行访问控制。**


决策引擎是以**规则引擎、模型计算、流程编排**为内核的业务决策中台，核心解决：业务策略频繁变更、规则散落在代码、策略效果无法量化、审批流程割裂四大痛点，覆盖信贷、支付风控、营销、保险、会员、订单审核等所有需要标准化自动判断的业务。

## 核心设计

决策引擎核心价值：把业务判断逻辑从代码抽离、可视化配置、统一调度规则 + 模型，支撑自动化、可运营、可迭代的标准化决策，覆盖金融、零售、政企、互联网、制造等领域

- 使用表达式和内置函数表示提取任意维度的特征
- 以流程编排为核心，编排不同的节点去刻画不同的业务流程
- 引入自定义节点设计可插拔的拓展体系
- 通过**数据、模型、流程的差异化配置**来适配各个场景

要为金融、营销、广告、防撞库等多种场景构建一个**通用的决策引擎**，关键在于设计一套 **“核心稳定、场景可插拔”** 的架构。这意味着引擎的核心是通用的，但能通过配置化方式快速适配不同业务。

它的实现思路可以概括为：**定义统一的“决策语言”**（元模型），构建**分层、模块化的技术架构**，并通过**数据、模型、流程的差异化配置**来适配各个场景。


为了满足不同的运营目标, 对抗多变的风险因素, 我们需要对动态对业务场景中的属性进行判断, 计算, 流程编排, 动态配置生效。为了满足这些目标，我们设计了以规则流程优先的设计模式来组织判定逻辑。同时提供以下功能来简化规则的书写，强化规则流程的关系，沉淀业务的核心逻辑。

- 表达式判断
- 流程编排
- 实时计算引擎
- 动态配置

### 1

“学习”现有的商业专业知识（在人类专家的头脑中）效率低下。它需要大量的时间和计算机资源。如果决策过程中存在复杂但确定性的因素，这些因素很容易理解，确实需要重新学习，那么这些因素可以更好地表示为由人类专家创建的决策表网络。即使这些并不完美，它们也可以通过机器学习得到改进，而不是从头开始学习一切。

有些逻辑，例如合规性，本质上是易变、复杂且定义明确的。将其作为机器学习模型的重点是不明智的，否则将需要频繁（且昂贵）的重新训练。

人类专家创建的决策表网络比机器学习模型具有更高的可解释性，尽管机器学习模型准确度高，但往往无法阐明其原理。自动化决策的许多方面都面临着越来越大的压力，需要解释其结果。例如，法规要求决策证明，换句话说，它们不存在种族、性别或其他受保护标准之间的偏见。在某些情况下，他们不得以任何方式使用受保护的标准。使用静态业务规则更容易实现这一点，而且更重要的是更容易证明这一点。

有时不可能训练机器学习模型具有与主题专家构建的决策模型相同的召回率和精确度，因为训练数据可能很嘈杂，并且训练过程本身并不完善。

### 2

构建通用决策引擎的核心思想是**“分离关注点”**：

1. **引擎关注“如何做决策”**：通过**决策流编排**和**规则执行**的核心能力，提供通用的决策框架。
2. **业务关注“决策什么”**：通过**可视化的配置**（数据、规则、模型、流程），让业务人员可以无需编码就能定义和调整策略[](https://blog.csdn.net/weixin_28679635/article/details/161128635)。

通过这种分层、模块化、配置驱动的架构，就能构建出一个既能满足金融风控的严谨、又能适应营销推荐的灵活、还能抵御撞库攻击的迅捷的强大通用决策引擎。

## 一、金融行业（最主流落地场景）

### 1. 信贷信用业务

1. 线上小额贷、消费贷授信审批
    
    整合征信、多头借贷、收入、负债、人行报告、行为分，自动输出：通过 / 拒绝 / 人工复核、授信额度、分期利率。
2. 信用卡申请审批、提额、降额、冻结
    
    新客准入策略、存量客户动态额度调整、逾期风险识别降额。
3. 企业经营贷、税贷、供应链贷风控决策
    
    企业开票、纳税、流水、法人风险、关联企业团伙风险综合判定。
4. 贷中风险监控、贷后催收分级
    
    贷中异动监测（频繁异地、大额取现）；贷后分档：短信催收、电话催收、上门、法务诉讼分级策略。

### 2. 支付反欺诈（交易决策）

- 线上支付、快捷支付、跨境支付实时风控：盗刷、伪卡、批量刷单、盗号支付拦截；
- 商户准入、商户风险评级、T+1 结算冻结策略；
- 充值、提现、转账限额动态管控。

### 3. 保险业务核保 & 理赔

1. 投保核保决策：健康告知、年龄职业风险、既往理赔记录，自动承保 / 加费 / 拒保；
2. 理赔自动审核：小额快速赔付、虚假理赔识别（重复报案、病史造假、团伙骗保）；
3. 车险保费自动定价：车型、出险记录、驾驶行为浮动费率计算。

### 4. 财富理财、资管

- 客户风险测评（保守 / 稳健 / 进取），匹配对应理财产品；
- 大额交易反洗钱（AML）：识别可疑交易、分拆转账、跨境大额，触发上报；
- 理财赎回限制、产品购买准入门槛管控。

## 二、互联网 / 电商 / 本地生活

### 1. 营销活动智能决策

- 优惠券、满减、红包发放准入：新老客、会员等级、历史核销、限购防羊毛党；
- 千人千面权益分配：高价值客户大额券、沉睡客户唤醒券；
- 活动防作弊：批量注册领券、机器刷活动、多账号薅补贴拦截。

### 2. 订单与售后决策

- 下单风控：黄牛囤货、虚假地址、批量下单拦截；
- 售后自动判定：无理由退款、质量问题赔付、恶意仅退款识别；
- 运费险自动定价、拒赔规则。

### 3. 账号安全 & 注册风控

- 注册 / 登录决策：异常设备、异地 IP、手机号黑名单，验证码强度、直接拦截；
- 账号分级管控：新号限制发帖、高风险账号禁言冻结；
- 实名认证校验决策。

## 三、政企政务场景

1. 政务事项智能审批
    
    营业执照办理、低保申请、公积金提取、人才补贴、落户资格自动初审，减少人工窗口；
2. 公共信用分级
    
    企业信用评级、个人失信判定、行业黑名单自动匹配；
3. 交通 / 城管 / 监管判别
    
    网约车司机准入、培训机构资质校验、违规企业预警判定；
4. 税务智能风控
    
    发票虚开预警、企业偷税漏税风险识别、优惠政策自动判定是否可享受。
