1. Inside the Store (`set` and `get`)

When you [create a store](https://github.com/pmndrs/zustand), these functions are passed to the initializer to manage internal logic. [[1](https://github.com/pmndrs/zustand), [2](https://medium.com/design-bootcamp/react-state-management-with-zustand-9ddc7d8181be)]

- **`set`**: Used to update the state. It shallowly merges the current state with the new values you provide.
- **`get`**: Used to read the current state value without triggering a re-render. This is helpful for logic that depends on the current value before updating.

2. Outside the Store (`setState` and `getState`)

If you need to interact with the store outside of React components (e.g., in a utility function or a regular JS file), you use the [Store API](https://beyondthecode.medium.com/zustand-a-guide-to-scalable-state-management-0186c4208e01). [[1](https://beyondthecode.medium.com/zustand-a-guide-to-scalable-state-management-0186c4208e01)]

- **`getState()`**: Returns a snapshot of the current state at that moment.
- **`setState()`**: Updates the state from outside the hook system. [[1](https://zustand.docs.pmnd.rs/reference/apis/create), [2](https://dev.to/muthu_aravindh_668229c96f/a-beginners-guide-to-state-management-in-react-with-zustand-28k5), [3](https://beyondthecode.medium.com/zustand-a-guide-to-scalable-state-management-0186c4208e01)]
