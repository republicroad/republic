
使用 **DuckDB** 进行营销拉新归因（Marketing Attribution），核心优势在于其轻量化、列式存储和高性能的窗口函数能力。特别是在处理海量用户行为（点击流）和多触点（First-Touch, Last-Touch, 线性等）时，无需昂贵的大数据套件即可实现实时归因与拉新分析。 [[1](https://medium.com/@kaushalsinh73/from-dataframes-to-engines-pandas-polars-or-duckdb-42411a9da2d1), [2](https://podcasts.apple.com/us/podcast/onboard/id1613083252?l=ko), [3](https://www.dench.com/blog/duckdb-marketing-analytics)]

以下是基于 DuckDB 的归因算法落地指南与常用 SQL 模板：

一、 归因算法分类与 SQL 实现（基于窗口函数）

用户拉新过程涉及多个触点（例如：看到广告 → 搜索 → 点击 → 注册）。假设我们的数据表名为 `marketing_touchpoints`，包含字段：`user_id`（用户ID）、`channel`（广告渠道）、`touch_time`（触点时间）、`event_type`（事件类型，如click/register）。 [[1](https://www.fxiaoke.com/crm/information-78885.html), [2](https://cloud.tencent.com/developer/article/2549137)]

## 1. 首次触点归因（First-Touch Attribution）

将 100% 的拉新功劳归于用户生命周期中的**第一个**渠道。 [[1](https://www.fxiaoke.com/crm/information-78885.html)]

```sql
WITH RankedTouches AS (
    SELECT 
        user_id,
        channel,
        ROW_NUMBER() OVER(PARTITION BY user_id ORDER BY touch_time ASC) as rn
    FROM marketing_touchpoints
    WHERE event_type = 'click'
)
SELECT 
    channel,
    COUNT(DISTINCT user_id) as new_users
FROM RankedTouches
WHERE rn = 1
GROUP BY channel;
```

Use code with caution.

## 2. 末次触点归因（Last-Touch Attribution）

将 100% 的拉新功劳归于用户**转化（注册）前最后一次**接触的渠道。 [[1](https://www.fxiaoke.com/crm/information-78885.html)]

```sql
WITH RankedTouches AS (
    SELECT 
        user_id,
        channel,
        ROW_NUMBER() OVER(PARTITION BY user_id ORDER BY touch_time DESC) as rn
    FROM marketing_touchpoints
    WHERE touch_time <= (
        -- 获取用户的注册时间
        SELECT MAX(touch_time) 
        FROM marketing_touchpoints t2 
        WHERE t2.user_id = marketing_touchpoints.user_id AND event_type = 'register'
    )
)
SELECT 
    channel,
    COUNT(DISTINCT user_id) as new_users
FROM RankedTouches
WHERE rn = 1
GROUP BY channel;
```

Use code with caution.

## 3. 线性归因（Linear Attribution）

将拉新功劳**平分**给该用户转化前所有的广告触点。如果用户有 4 个触点，每个触点获得 0.25 个新用户权重。 [[1](https://www.reddit.com/r/BusinessIntelligence/comments/1ig4a6n/how_to_build_an_attribution_model_i_need_examples/)]

```sql
WITH ChannelCounts AS (
    SELECT 
        user_id,
        channel,
        -- 计算该用户总共有多少次点击
        COUNT(*) OVER(PARTITION BY user_id) as total_touches
    FROM marketing_touchpoints
    WHERE event_type = 'click'
)
SELECT 
    channel,
    -- 每个渠道按比例累加权重
    SUM(1.0 / total_touches) as attributed_new_users
FROM ChannelCounts
GROUP BY channel;
```
